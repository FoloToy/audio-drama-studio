# 儿童音频短剧创作 Agent 平台 PRD

## 1. 产品概述

### 1.1 产品名称

儿童音频短剧创作 Agent 平台

### 1.2 产品形态

Web 应用。

### 1.3 产品定位

本产品是一个面向儿童内容创作者的 Web 版 AI 音频短剧创作平台。

用户可以新建音频短剧项目，上传原著、粘贴故事文本或输入故事主题，选择目标年龄段、集数、单集时长、故事风格和音频形式。系统通过多个 Agent 自动完成素材解析、故事拆集、适龄改编、剧本生成、角色声音推荐、音频生成与导出。用户在关键节点进行人工确认和修改，最终得到适合儿童收听的多集音频短剧。

---

## 2. 背景与痛点

儿童音频内容生产存在以下痛点：

1. 原著故事不适合直接朗读给儿童。
2. 不同年龄段儿童的理解能力、注意力、语言接受度差异很大。
3. 传统音频短剧制作流程长，需要编剧、审校、配音、音效、后期等多个环节。
4. 现有 TTS 工具通常只能“读文本”，不能完成“故事改编”和“短剧创作”。
5. 内容团队需要持续、批量、稳定地产出适合儿童的音频故事。
6. AI 生成内容需要人工审核，尤其儿童内容必须控制暴力、恐怖、成人化和价值观风险。

---

## 3. 产品目标

### 3.1 核心目标

构建一个 Web 版音频短剧创作平台，使用户能够通过简单配置，快速完成儿童音频短剧的创作、审核、配音和导出。

### 3.2 MVP 闭环

```text
新建项目
→ 输入故事素材
→ 选择目标年龄段
→ 选择集数与风格
→ Agent 生成分集大纲
→ 用户确认大纲
→ Agent 生成剧本
→ 用户审核与修改剧本
→ 选择角色声音
→ 生成音频
→ 试听与导出
```

### 3.3 MVP 非目标

MVP 阶段暂不重点实现：

1. 多人协作审稿。
2. 完整版权管理系统。
3. 复杂音频 DAW 式多轨编辑。
4. 大规模内容分发平台。
5. 商业化结算系统。
6. 未授权真人声音克隆。
7. 移动端完整创作体验。
8. 自动发布到第三方平台。

## 4. 用户角色

### 4.1 儿童内容编辑

主要使用者，负责输入故事、确认大纲、修改剧本、审核内容、选择声音和导出音频。

### 4.2 编剧 / 内容策划

负责故事结构与剧本质量，重点关注分集结构、人物对白、旁白比例、剧情节奏和结尾悬念。

### 4.3 音频制作人员

负责配音、试听和导出，重点关注角色声音一致性、局部重生成、语速、停顿、音效和 BGM。

### 4.4 管理员

负责声音库、风格模板、儿童安全规则、项目记录和系统日志管理。

---

## 5. 核心场景

### 5.1 上传《西游记》原著，生成儿童短剧

用户新建项目，上传《西游记》文本，选择 5-8 岁、10 集、经典儿童广播故事风、旁白 + 多角色短剧。系统解析原著，生成 10 集大纲，用户确认后生成剧本，为角色绑定声音，生成并导出音频。

### 5.2 输入“三国火烧赤壁”，生成 3 集短剧

用户输入“火烧赤壁”，选择 5-8 岁、3 集、孙敬修式儿童故事感。系统生成 3 集大纲和剧本，用户审核后生成多角色音频。

### 5.3 原创睡前故事系列

用户输入“小熊不想睡觉”，选择 3-5 岁、5 集、睡前陪伴风。系统生成低刺激、温柔、慢节奏的睡前音频故事。

## 6. 年龄段策略

### 6.1 3-5 岁

要求：句子短、情节简单、人物少、情绪温和、避免死亡/战争/恐怖/复杂阴谋，多使用重复、声音、动作和简单互动。单集建议 3-5 分钟。

### 6.2 5-8 岁

要求：可以有简单冲突、清晰因果、幽默角色互动和轻量冒险，但要弱化暴力，价值表达要明确。单集建议 5-8 分钟。

### 6.3 8-12 岁

要求：可以保留更多原著信息、复杂人物动机、策略、选择、责任和成长主题，但仍需避免血腥、成人权谋和过度阴暗表达。单集建议 8-12 分钟。

---

## 7. 故事风格模板

MVP 支持以下风格：

1. 经典儿童广播故事风
2. 孙敬修式儿童故事感
3. 睡前陪伴风
4. 冒险喜剧风
5. 国学启蒙风
6. 温柔治愈风

“孙敬修式儿童故事感”在产品中可描述为“经典老一辈儿童故事讲述感”，特点是亲切、慢节奏、画面感强、像长辈讲故事，不夸张、不吵闹。

## 8. 核心功能需求

### 8.1 项目管理

项目列表展示项目名称、封面、项目类型、目标年龄、集数、当前状态、最近更新时间、创建人和操作按钮。

项目状态：草稿、素材解析中、待确认大纲、剧本生成中、待审核剧本、待选择声音、音频生成中、待试听、已完成、已导出。

支持操作：新建项目、打开项目、复制项目、删除项目、搜索项目、按状态筛选项目。

### 8.2 新建项目

必填字段：项目名称、内容来源、目标年龄、计划集数、单集时长、故事风格、音频形式。

可选字段：项目简介、原著忠实度、角色数量上限、是否自动生成音效、是否自动生成 BGM、是否启用儿童安全增强审核。

### 8.3 内容输入

MVP 支持：粘贴文本、上传 TXT / Markdown、输入故事主题。

后续支持：PDF、EPUB、Word、素材库选择、已有项目复用。

### 8.4 素材解析

输出：文本摘要、主要角色、主要情节、可改编故事单元、风险内容标记、建议分集方向、推荐目标年龄、是否适合儿童改编。

### 8.5 故事拆集

根据素材、年龄段、集数、单集时长和风格，生成分集大纲。

每集包含：集数、标题、本集摘要、开场钩子、核心冲突、主要角色、情绪曲线、教育价值、结尾悬念、风险等级、建议时长。

### 8.6 适龄改编

根据年龄段自动处理语言难度、情节复杂度、人物数量、冲突强度、暴力内容、恐怖内容、成人化内容、历史背景解释、价值观表达方式和情绪刺激强度。

### 8.7 剧本生成

根据确认后的分集大纲生成结构化音频剧本。

每集剧本包含：集标题、目标年龄、预计时长、角色表、分段剧本、旁白、角色台词、音效提示、BGM 提示、情绪标签、语速建议、停顿建议和本集结尾。

剧本块类型：旁白、台词、音效、BGM、停顿、转场。

### 8.8 剧本编辑

用户可以修改台词、修改旁白、删除剧本块、新增剧本块、修改角色、修改情绪标签、修改语速标签、添加音效、删除音效、重新生成某一段、重新生成整集、锁定某一段、标记为已审核。

### 8.9 儿童安全审核

审核维度：暴力描写、恐怖惊吓、成人化内容、性暗示、歧视与羞辱、粗口脏话、过度惊悚、消极价值观、鼓励危险行为、不适合儿童的历史表达、过度迷信表达、不适合低龄儿童的死亡表达。

风险等级：低风险、中风险、高风险、禁止生成。

每个风险提示包含：风险位置、风险类型、风险说明、建议修改方案、一键改写按钮。

### 8.10 角色管理

Agent 自动识别角色。每个角色包含角色名称、角色类型、性格、年龄感、声音建议、出现集数、角色关系、口头禅、禁用表达。

用户可以锁定角色设定。锁定后，重新生成时不得随意改变角色性格和声音。

### 8.11 声音库与声音绑定

声音类型包括旁白声音、角色声音、儿童角色声音、动物角色声音、老人声音、品牌 IP 声音和授权真人声音。

每个声音包含声音名称、声音 ID、性别感、年龄感、音色描述、情绪支持、语速范围、适合年龄段、适合场景、授权状态、示例音频、是否可商用。

MVP 不支持未授权真人声音克隆。

### 8.12 音频生成

支持按整集生成和按句重新生成。

每集音频包含旁白音频、角色对白、停顿、音效、BGM、片头、片尾。

导出格式 MVP 支持 MP3，后续支持 WAV、分轨 WAV、SRT 字幕、剧本文档和 JSON 工程文件。

## 11. Agent 架构概览

MVP 需要以下 Agent：

1. 素材解析 Agent
2. 故事拆集 Agent
3. 适龄改编 Agent
4. 风格转换 Agent
5. 剧本生成 Agent
6. 儿童安全审核 Agent
7. 角色识别 Agent
8. 声音匹配 Agent
9. 音频生成 Agent
10. 混音 Agent

工作链路：

```text
用户输入素材
→ 素材解析 Agent
→ 儿童安全审核 Agent
→ 故事拆集 Agent
→ 适龄改编 Agent
→ 风格转换 Agent
→ 剧本生成 Agent
→ 儿童安全审核 Agent
→ 用户审核
→ 角色识别 Agent
→ 声音匹配 Agent
→ 用户确认声音
→ 音频生成 Agent
→ 混音 Agent
→ 用户试听
→ 导出
```

## 12. 验收标准

### 12.1 新建项目

用户可以完成项目名称、内容来源、故事文本、目标年龄、集数、风格配置，并成功进入素材解析阶段。

### 12.2 分集大纲

系统可以根据用户设置生成指定集数的大纲。每集必须包含标题、摘要、开场钩子、核心冲突、主要角色、教育价值、结尾悬念和风险等级。

### 12.3 剧本生成

系统可以根据确认后的大纲生成结构化剧本。剧本必须包含旁白、角色台词、音效提示、BGM 提示、情绪标签、语速建议和预计时长。

### 12.4 剧本编辑

用户可以修改台词、旁白、新增或删除剧本块、重新生成局部内容，并标记剧本为已审核。

### 12.5 声音选择

用户可以查看角色列表、系统推荐声音、试听声音样例、为角色绑定声音并锁定。

### 12.6 音频生成

系统可以根据剧本生成单集音频，区分不同角色声音，插入基础音效和 BGM，支持试听和导出 MP3。

### 12.7 安全审核

系统可以识别明显不适合儿童的内容，给出风险等级、风险位置和修改建议，并阻止高风险内容直接进入音频生成。
